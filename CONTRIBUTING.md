# Contributing

Thank you for investing time in improving the Cybersecurity Resume Builder. Please follow these guidelines:

## Development Workflow
1. Fork or branch from `work`.
2. Install dependencies with `npm install`.
3. Add or update tests when changing functionality.
4. Run the test suite with `npm test` before opening a PR.
5. Keep commits focused and well-described.

## Code Style
- ES modules with clear, self-documenting names.
- Prefer pure functions and small modules.
- Avoid try/catch around imports.
- Document exported functions with short comments when behavior is non-obvious.

## Security & Privacy
- Do not store personal data in the repository.
- Keep generated examples fictional unless explicit data is provided.
- Follow the guidance in `SECURITY_POLICY.md` for reporting vulnerabilities.

## Communication
- Open issues for feature requests or bugs with reproduction steps.
- Keep discussions respectful and aligned with the Code of Conduct.
